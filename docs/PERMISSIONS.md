# Microsoft Graph permissions required by RKSolutions

This document lists the Microsoft Graph API permissions (scopes) required by each cmdlet. It was generated by reviewing the `RequiredScopes` (or equivalent) in each script under `module/Public/` and `module/Private/` where connection scopes are defined.

Use these when registering an app in Azure AD / Entra ID or when calling `Connect-RKGraph -RequiredScopes` to limit scopes to only what you need.

## Notes

- **Application vs Delegated:** These are typically **Application** or **Delegated** depending on whether you connect with client secret/certificate (application) or interactive/device code (delegated). Configure the same permission names in the Azure app registration under Microsoft Graph.
- **Mail.Send:** Required only if you use the `-SendEmail` / `-Recipient` parameters on the report cmdlets. Omit from `-RequiredScopes` if you do not send email.
- **Least privilege:** To run only one report, pass that cmdlet’s scopes to `Connect-RKGraph -RequiredScopes` instead of the default union.
